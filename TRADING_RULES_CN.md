## 🔷 Uniswap V3 ETH/USDC LP 交易规则

### 规则 #1: 仓位管理规则

**适用脚本**: `quick_eth_usdc_lp.py`

#### 1.1 开仓条件
- ✅ **必须满足**：钱包中有足够的 ETH 和 USDC
- ✅ **必须满足**：额外预留 0.01-0.05 ETH 作为 gas 费
- ✅ **必须满足**：Gateway 连接正常
- ✅ **建议**：首次交易使用最小金额测试

#### 1.2 仓位大小规则
```yaml
# 保守型（适合新手）
entry_amount_eth: 0.05      # 不超过总 ETH 的 10%
entry_amount_usdc: 150      # 不超过总 USDC 的 10%

# 平衡型（有经验）
entry_amount_eth: 0.1       # 不超过总 ETH 的 20%
entry_amount_usdc: 300      # 不超过总 USDC 的 20%

# 激进型（专业用户）
entry_amount_eth: 0.2       # 不超过总 ETH 的 30%
entry_amount_usdc: 600      # 不超过总 USDC 的 30%
```

**硬性规则**：
- ❌ 禁止：单次仓位超过总资金的 50%
- ❌ 禁止：不预留 gas 费就开仓
- ❌ 禁止：在价格剧烈波动（±10% 在 1 小时内）时开仓

---

### 规则 #2: 价格区间设置规则

#### 2.1 根据市场波动率选择区间

| 市场状况 | 波动率 | range_lower_pct | range_upper_pct | 预期 APY |
|---------|--------|-----------------|-----------------|----------|
| 极度平稳 | <2%/天 | 1% | 1% | 50-100% |
| 平稳 | 2-5%/天 | 2-3% | 2-3% | 30-60% |
| 正常 | 5-10%/天 | 5% | 5% | 20-40% |
| 波动 | 10-20%/天 | 7-10% | 7-10% | 10-25% |
| 剧烈波动 | >20%/天 | 15%+ | 15%+ | <10% |

#### 2.2 区间设置规则
- ✅ **必须**：lower_pct 和 upper_pct 总和不能小于 2%
- ✅ **建议**：新手使用 5-10% 区间
- ⚠️ **警告**：<2% 区间需要频繁监控和调整
- ❌ **禁止**：设置 <0.5% 的极窄区间（除非 24/7 监控）

### 规则 #3: 风险控制规则

#### 3.1 止损规则（必须遵守）

```yaml
# 强制止损设置
stop_loss_pct: 10          # 价格下跌 10% 自动平仓

# 根据仓位大小调整
小仓位 (<$500):   stop_loss_pct: 15
中仓位 ($500-2000): stop_loss_pct: 10
大仓位 (>$2000):    stop_loss_pct: 5
```

**触发条件**：
- 价格相对开仓价下跌超过设定百分比
- 立即执行，不等待恢复

#### 3.2 无常损失 (IL) 管理规则

**自动计算 IL**：
```
IL = (price_now / price_entry)^0.5 * 2 / (1 + price_now/price_entry) - 1
```

**响应策略**：
- IL > 5%: 📝 记录，继续观察
- IL > 10%: ⚠️ 评估是否调整区间
- IL > 20%: 🚨 强烈建议平仓或调整策略
- IL > 30%: ❌ 必须平仓（除非有明确的恢复理由）

### 规则 #4: 监控和复核规则

#### 4.1 定期检查（自动化）

```yaml
check_interval: 15         # 每 15 秒检查一次

# 检查项目：
✓ 当前价格是否在区间内
✓ 无常损失是否超标
✓ 累计手续费收益
✓ 止损条件
✓ Gas 费余额
```

#### 4.2 人工复核（必须执行）

| 频率 | 检查内容 | 必要操作 |
|-----|---------|---------|
| 每天 | 查看 `status` 命令输出 | 确认仓位健康 |
| 每周 | 查看 `history` 历史记录 | 计算实际收益率 |
| 每月 | 审核策略参数 | 根据表现调整 |

**强制复核触发条件**：
- 🚨 价格单日波动 >15%
- 🚨 连续 3 天价格在区间外
- 🚨 累计 IL >15%
- 🚨 ETH 价格突破关键支撑/阻力位

#### 5.1 根据资金量选择网络


**禁止事项**：
- ❌ 有未平仓位时切换网络
- ❌ 在不同网络上同时运行相同策略
- ❌ 混淆不同网络的代币地址

**允许操作**：
- ✅ 平仓后切换到低 gas 网络
- ✅ 在测试网测试后切换到主网
- ✅ 根据 gas 价格动态选择网络

## 🔷 风险管理规则

### 规则 #8: 紧急止损规则

#### 8.1 自动触发条件

系统会在以下情况自动平仓：

1. **价格止损**
   - 价格跌破止损线 (`stop_loss_pct`)
   
2. **时间止损**
   - 连续 24 小时价格在区间外
   
3. **IL 止损**
   - 无常损失超过 25%

#### 8.2 手动止损指南

**立即平仓情况**：
- 🚨 黑天鹅事件（如交易所被黑、监管新闻）
- 🚨 发现代码漏洞或异常行为
- 🚨 当前 IL >30% 且趋势继续恶化
- 🚨 需要资金用于其他用途

**如何手动平仓**：
```bash
# 方法 1：在 Hummingbot 中
stop

# 方法 2：直接在 Uniswap 界面
访问：https://app.uniswap.org
连接钱包 → Pools → Close Position
```

## 🔷 紧急情况处理规则

### 规则 #10: 故障应对规则

#### 10.1 系统故障类型

**A. 网络连接中断**
```yaml
症状：无法获取价格、无法提交交易
处理：
  1. 检查网络连接：ping google.com
  2. 检查 Gateway 状态：gateway status
  3. 重启 Gateway：gateway restart
  4. 如果紧急，使用 Uniswap 界面手动操作
```

**B. Gateway 错误**
```yaml
症状：交易失败、余额显示错误
处理：
  1. 查看日志：tail -f logs/logs_script.log
  2. 重启 Gateway：docker-compose restart gateway
  3. 重新连接钱包：gateway connect ethereum
  4. 验证连接：gateway connector-tokens uniswap ethereum
```

**C. 交易卡住不确认**
```yaml
症状：交易已提交但长时间不确认
处理：
  1. 在区块浏览器查看 tx：https://etherscan.io
  2. 如果 pending >30 分钟，考虑加速
  3. 使用 MetaMask 的 "Speed Up" 功能
  4. 最后手段：取消交易（可能损失 gas）
```

#### 10.2 紧急联系方式

```

  → #support 频道
  → #gateway 频道

紧急文档：
  → https://hummingbot.org/gateway/troubleshooting
```

## 🔷 性能评估规则

### 规则 #11: 策略评估标准

#### 11.1 每日评估指标

```python
# 每日必须计算
日收益率 = (当前价值 + 累计手续费 - 初始投入) / 初始投入 / 运行天数

# 目标基准
优秀：>0.1% 每天 (>36.5% APY)
良好：0.05-0.1% 每天 (18-36% APY)
及格：0.03-0.05% 每天 (11-18% APY)
不及格：<0.03% 每天 (<11% APY)
```

#### 11.2 周度评估

**每周必须回答的问题**：
1. 实际收益率 vs 预期？
2. 价格是否频繁出区间？
3. Gas 费占总收益的比例？（应 <20%）
4. 无常损失是否在可控范围？（应 <10%）

**调整决策**：
- ✅ 收益达标且 IL 可控：继续当前策略
- ⚠️ 收益不达标：调整区间或止损参数
- ❌ IL 持续扩大：考虑暂停策略

## 🎓 新手特别规则

如果您是第一次使用 LP 策略，请额外遵守：

### 新手安全规则

1. **从测试网开始**
   ```yaml
   network: goerli  # 或其他测试网
   ```

2. **使用最小金额**
   ```yaml
   entry_amount_eth: 0.01    # 仅 $20-30
   entry_amount_usdc: 30
   ```

3. **使用保守参数**
   ```yaml
   range_lower_pct: 10       # 宽区间
   range_upper_pct: 10
   stop_loss_pct: 15         # 宽止损
   ```

4. **强制学习期**
   - 至少运行 1 周测试
   - 至少完成 3 次完整开仓/平仓周期
   - 至少手动记录 5 次交易日志
   - 至少经历 1 次价格出区间情况

5. **禁止事项**
   - ❌ 第一次就投入大额资金
   - ❌ 使用激进参数（窄区间、紧止损）
   - ❌ 同时运行多个策略
   - ❌ 在不理解的情况下修改代码
